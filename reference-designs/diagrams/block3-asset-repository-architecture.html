DCIM Block 3 — Asset Repository

Reference Architecture • CRUD API • Bulk Import • Reconciliation • Audit Trail • Enrichment API • Redis Cache

PostgreSQL Schema (4 tables)

  • • asset — Core asset data (primary)
  • • asset_location — Physical location hierarchy
  • • asset_financial — Depreciation & book value
  • • asset_contract — Warranty, SLA, contracts
  • • asset_audit_log — Immutable change history

API Endpoints (6 groups)

  • • CRUD — POST/GET/PUT/DELETE assets
  • • Bulk Import — CSV/JSON upsert
  • • Search — Full-text with trigram
  • • Enrichment — CI → Asset (cached)
  • • Audit Trail — Immutable history

Service Layer

  • • Validation — Schema + uniqueness
  • • Reconciliation — CMDB ↔ Asset sync
  • • Lifecycle Manager — State transitions
  • • Depreciation Calculator — Monthly batch
  • • Cache Service — Redis 15min TTL

Security & Audit

  • • RBAC — 5 roles (viewer → auditor)
  • • Immutable audit log (append-only)
  • • Partitioned monthly (7yr retention)
  • • Rate limiting per endpoint
  • • TLS 1.2+ all traffic

Data Quality

  • • Asset_ID uniqueness enforced
  • • Serial + manufacturer unique
  • • 8 mandatory fields
  • • Referential integrity (location, contract)
  • • Daily quality scorecard view

Performance Targets

  • • Enrichment p99 < 50ms
  • • CRUD p99 < 100ms
  • • List p99 < 200ms
  • • Bulk import < 30s (500 records)
  • • Cache hit ratio > 80%
