Verdict: Pilih Kafka Jika

  • • Volume > 5000 events/detik
  • • Butuh replay > 24 jam (forensic investigation)
  • • Multi-consumer (ES + Workflow + Analytics + Compliance)
  • • Compliance audit butuh complete event trail
  • • DCIM production — data loss = blind spot keamanan

Verdict: Skip Kafka Jika

  • • Volume rendah (< 2000 events/detik)
  • • Single consumer (ES saja)
  • • Dev/staging environment
  • • Budget sangat terbatas
  • • Tim kecil, tidak ada dedicated Kafka ops

Rekomendasi Hermes

  • • DCIM Production → Kafka (data integrity > cost)
  • • Dev/Staging → No-Kafka (faster bootstrap)
  • • Middle ground: Start no-Kafka, migrate when volume justifies
  • • Hybrid: Kafka untuk SIEM, NiFi untuk DI&I non-security
  • • Risk: No-Kafka di production = single failure point di NiFi
