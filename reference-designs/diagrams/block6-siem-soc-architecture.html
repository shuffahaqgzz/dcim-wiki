DCIM Block 6 — SIEM/SOC

Reference Architecture • Wazuh • Kafka • Elasticsearch • Correlation Engine • Incident Response • CIS Compliance • SOC API

Ingestion Pipeline

  • • Wazuh agents on all servers
  • • Syslog → NiFi → Kafka → Elasticsearch
  • • ~130K events/day normalized
  • • Real-time (sub-5s latency)
  • • Auto-enrich with asset/CI info

Correlation Engine (10 rules)

  • • Brute force / Credential stuffing
  • • Privilege escalation / Lateral movement
  • • File tampering / Malware detection
  • • Data exfiltration / Config change
  • • Threshold + Pattern + Cross-source

Incident Response

  • • 6-state lifecycle (created→closed)
  • • SLA: Critical <15min, High <30min
  • • Auto-isolate (malware), auto-block IP
  • • Integration with Workflow Automation
  • • Forensics data collection

CIS Compliance

  • • 6 benchmarks (Linux, Win, PG, Docker, K8s)
  • • Automated daily scans
  • • Gap analysis + remediation commands
  • • Compliance score tracking
  • • 7-year report retention

SOC API (12 endpoints)

  • • Events query (severity, category, IP)
  • • Alerts management
  • • Incidents CRUD + assign + resolve
  • • Compliance reports + export
  • • Dashboard summary

Security & Retention

  • • TLS 1.2+ all traffic
  • • AES-256 encryption at rest
  • • 4 RBAC roles (admin→viewer)
  • • Events: 90d, Alerts: 180d, Incidents: ∞
  • • Compliance: 7-year retention
