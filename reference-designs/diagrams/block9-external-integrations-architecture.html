Reference Architecture • Adapter Pattern • ITSM • ERP • DMS • NMS • Cloud (AWS/GCP/Azure) • Health Monitoring

Adapter Framework

  • • Base adapter: pull/push/normalize/sync
  • • Circuit breaker (5 failures → open)
  • • Normalizer (YAML mapping configs)
  • • Error handler (DLQ + retry backoff)
  • • Health check per connector

ITSM Connectors (2)

  • • ServiceNow: OAuth2, bidirectional
  • • Jira: API Key, bidirectional
  • • Auto-create tickets from alerts
  • • Status sync (workflow ↔ ticket)
  • • SLA tracking integration

ERP Connectors (2)

  • • SAP: asset master + financial data
  • • Oracle: procurement + inventory
  • • Batch sync (daily)
  • • Depreciation + book value sync
  • • PO number cross-reference

DMS + NMS Connectors (2)

  • • DMS: document linking to CIs/assets
  • • DMS: SOP/manual attachment
  • • NMS: SNMP device discovery
  • • NMS: topology import
  • • NMS: performance metrics polling

Cloud Connectors (3)

  • • AWS: EC2, RDS, S3, CloudWatch
  • • GCP: Compute Engine, Cloud SQL
  • • Azure: VMs, SQL, Monitor
  • • Resource discovery + sync
  • • Tag-based CI mapping

Health Monitoring

  • • Connectivity test (5min interval)
  • • Sync status tracking
  • • Circuit breaker state
  • • Error rate monitoring
  • • Alert: down, stale, circuit_open
