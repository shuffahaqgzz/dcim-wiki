DCIM Block 8 — Workflow Automation

Reference Architecture • State Machine • ITSM • Approvals • Runbook • Auto-Remediation • Escalation • n8n/Temporal

State Machine (10 states)

  • • pending → in_progress → waiting_approval
  • • approved → executing → completed/failed
  • • rejected, rollback, cancelled
  • • Validated transitions only
  • • PostgreSQL state persistence

ITSM Integration

  • • ServiceNow + Jira bidirectional
  • • Auto-create tickets from alerts
  • • Priority mapping (P1→Critical)
  • • Status sync (workflow ↔ ticket)
  • • SLA tracking

Approval Workflows

  • • Multi-level chains (1-3 levels)
  • • Role-based approvers
  • • Timeout escalation
  • • Batch approval support
  • • 4 chain types (standard→critical)

Runbook Engine

  • • Automated + manual steps
  • • Conditional logic (on_failure)
  • • Template variables ({{workflow.*}})
  • • Rollback procedures
  • • Audit trail per step

Auto-Remediation + Escalation

  • • Safety: blast radius, approval, rate limit
    • • 4 remediation rules defined
  • • Rollback on failure
  • • 4-level escalation matrix
  • • On-call schedule integration

n8n/Temporal + Notifications

  • • n8n: visual, simple automations
  • • Temporal: complex, long-running
  • • 6 notification channels
  • • Webhook + REST API triggers
  • • 7 workflow types defined
