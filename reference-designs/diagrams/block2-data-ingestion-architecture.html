DCIM Block 2 — Data Ingestion & Integration

Reference Architecture • NiFi Flows • Kafka Pipeline • Validation • Enrichment • DLQ • Lineage • Connectors

Kafka Pipeline

  • • 10 topics across pipeline stages
  • • dcim.events.raw → validated → enriched
  • • dcim.events.dlq for failed events
  • • RF=3, min.insync.replicas=2
  • • Consumer groups per processing stage

NiFi Flows (5 pipelines)

  • • BMS/EPMS — REST/SNMP ingestion
  • • NMS — SNMP v2c/v3 interface polling
  • • Server/Storage — SSH metrics collection
  • • Cloud/VM — vSphere/AWS API
  • • Access Control — Badge file drops

Validation & DLQ

  • • JSON Schema validation (mandatory)
  • • Type, range, format checks
  • • Duplicate detection (Redis, 5min window)
  • • DLQ with retry (exponential backoff)
  • • Archive to S3 after max retries

Enrichment

  • • CI lookup from CMDB (5min cache)
  • • Location from Asset Repository
  • • Priority assignment (P1-P4 rules)
  • • Impact scoring (0-10)
  • • p99 latency < 50ms

Data Lineage

  • • 5-stage tracking (ingest → store)
  • • PostgreSQL storage (partitioned monthly)
  • • Error stage + message capture
  • • Processing duration metrics
  • • Daily quality scorecard view

Connectors & Security

  • • ServiceNow/Jira (ITSM) — bidirectional
  • • SAP/Oracle (ERP) — asset financial
  • • Custom DMS — documents
  • • Circuit breaker per connector
  • • TLS 1.2+ • OAuth2 • Kafka ACLs
