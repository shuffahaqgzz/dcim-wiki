DCIM Staging & Production Architecture

Dual-Environment Deployment with CI/CD Pipeline, DR & Security Hardening

Staging Environment

  • • VLAN 100-109 (3 VLANs)
  • • Single-node components
  • • 12 vCPU, 22GB RAM total
  • • 300GB SSD storage
  • • Synthetic/masked data only
  • • Basic monitoring (Email)
  • • Developer VPN access
  • • RPO: 24h, RTO: 8h

Production Environment

  • • VLAN 200-209 (4 VLANs)
  • • HA clusters (3-node each)
  • • 52 vCPU, 184GB RAM total
  • • 2.9TB SSD storage
  • • Live production data
  • • HA monitoring (PagerDuty)
  • • Jump host + Vault
  • • RPO: 1h, RTO: 4h

Deployment Pipeline

  • • Git → CI → Build → Staging
  • • Auto-deploy to staging
  • • Manual approval gate
  • • Blue-green to production
  • • Canary for critical services
  • • Instant rollback capability
  • • Promotion criteria enforced
  • • ArgoCD for GitOps

Disaster Recovery

  • • Production: Cross-site replication
  • • Continuous WAL archiving
  • • Hourly snapshots
  • • Kafka RF=3 for durability
  • • ES replication factor 3
  • • Vault auto-unseal
  • • IaC for fast rebuild
  • • DR runbook documented

Security Hardening

  • • Network segmentation (VLAN)
  • • TLS 1.2+ everywhere
  • • Vault for secrets (HA)
  • • Audit logging (all API)
  • • Rate limiting (API GW)
  • • RBAC + least privilege
  • • AES-256 encryption at rest
  • • DDoS protection

Monitoring Stack

  • • Prometheus Federation (Prod)
  • • Grafana HA (Prod)
  • • Multi-channel alerts
  • • Filebeat → Kafka → ES
  • • Jaeger APM tracing
  • • Blackbox exporter
  • • P1: PagerDuty + SMS
  • • P4: Daily digest
