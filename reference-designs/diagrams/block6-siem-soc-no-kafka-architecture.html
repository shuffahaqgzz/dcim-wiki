DCIM SIEM/SOC — No-Kafka Architecture

Direct pipeline: Wazuh → Syslog → NiFi → Elasticsearch · NiFi sebagai buffer & fan-out

Pipeline Tanpa Kafka

  • • Wazuh → Syslog Collector :514
  • • Syslog → NiFi (parse, validate, normalize, enrich)
  • • NiFi → Elasticsearch (REST Bulk API)
  • • NiFi FlowFile Repo = write-ahead log (limited replay)
  • • NiFi provenance = event lineage

Trade-off: Apa yang Hilang

  • • ❌ Replay window: NiFi provenance = hours, Kafka = 7-90 hari
  • • ❌ Fan-out: NiFi 1-to-N via OutputPort, tapi stateful
  • • ❌ Backpressure: NiFi queue bounded, ES burst bisa overwhelming
  • • ❌ Durability: NiFi repo di local disk, Kafka = replicated
  • • ❌ Ordering: NiFi flow = serial, Kafka partition = parallel

Kapan Pakai Arsitektur Ini

  • • Volume rendah-sedang (< 5000 events/detik)
  • • Tidak butuh replay > 24 jam
  • • Single consumer ES saja (tidak multi-consumer)
  • • Tim kecil, infra sederhana
  • • Budget / resource terbatas
