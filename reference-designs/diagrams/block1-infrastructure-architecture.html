DCIM Block 1 — Infrastructure Provisioning

Reference Architecture • PostgreSQL 16 • Redis 7 • Kafka 3.x • NiFi • Elasticsearch 8.x • Prometheus • Grafana • Vault

Data Layer

  • • PostgreSQL 16 — Primary + Replica (streaming replication)
  • • Redis 7 — Primary + Replica + 3 Sentinel nodes
  • • Elasticsearch 8.x — 3-node cluster, ILM policy
  • • PgBouncer — Connection pooling (500 max clients)

Messaging Layer

  • • Kafka 3.x — 3 brokers, KRaft (no ZooKeeper)
  • • 9 topics — RF=3, min.insync.replicas=2
  • • NiFi 1.x — Visual flow orchestration
  • • DLQ handling with retry mechanism

Security & Secrets

  • • Vault — 3-node HA, dynamic DB credentials
  • • TLS 1.2+ everywhere (all components)
  • • RBAC — SCRAM-SHA-256 (PG), ACL (Redis)
  • • Audit logging on all components

Monitoring & Observability

  • • Prometheus — 15s scrape, 30-day retention
  • • Grafana — 14 dashboards (NOC/SOC/Facilities)
  • • Alertmanager — Slack + Email + PagerDuty
  • • Node Exporter — CPU, memory, disk, network

Network & Deployment

  • • 3 VLANs — Management, Data, DMZ
  • • Firewall — Default deny, least privilege
  • • Docker Compose (dev) / K8s (prod)
  • • ~22 vCPU, ~80 GB RAM, ~1 TB storage minimum

Backup & Recovery

  • • PostgreSQL — pg_basebackup + WAL (RPO: 5min)
  • • Redis — RDB + AOF (RPO: 1s)
  • • Kafka — Topic snapshots (RPO: 1h)
  • • ES — Snapshot API, 30-day retention
