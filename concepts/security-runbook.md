# Security Runbook

Security procedures untuk DCIM platform.

## Incident Response

### 1. Detection
- Alert from [[siem-soc]]
- User report
- Automated detection

### 2. Triage
- Severity classification (S1-S4)
- Impact assessment
- Containment decision

### 3. Containment
- Isolate affected systems
- Preserve evidence
- Notify stakeholders

### 4. Investigation
- Log analysis
- Timeline reconstruction
- Root cause identification

### 5. Eradication
- Remove threat
- Patch vulnerabilities
- Update controls

### 6. Recovery
- Restore systems
- Verify integrity
- Monitor for recurrence

### 7. Post-Incident
- Lessons learned
- Process improvement
- Documentation update

## Vulnerability Management
- Regular scanning
- Patch management
- Configuration hardening
- CIS benchmark compliance

## Access Management
- User provisioning/deprovisioning
- Access review
- Privileged access management
- MFA enforcement
