## 7. Architecture Pattern Assessment

### 7.1 Current Architecture (v4.2)

```
"Telegraf-centric Pipeline"
┌─────────────────────────────────────────────────────────────┐
│  Telegraf (Collection) → Kafka → Normalizer → NiFi (Enrich) │
│       ↓                                    ↓                 │
│  5 collectors                     Enrichment API + Redis     │
│       ↓                                    ↓                 │
│  6 raw topics                     enriched → 3 consumers     │
│                                            ↓                 │
│                                   PG / ES / iTop             │
└─────────────────────────────────────────────────────────────┘
```

**Characteristics:**
- ✅ Simple, single-host deployment
- ✅ Telegraf handles all protocols
- ✅ NiFi used only for enrichment (lightweight)
- ✅ Good observability (JSON logging + Filebeat)
- ✅ AI-ready (archive + interface)
- ⚠️ Single Kafka broker (no HA)
- ⚠️ No schema registry
- ⚠️ No real-time processing

### 7.2 Reference Architecture (DCIM-Wiki)

```
"NiFi-centric Pipeline"
┌─────────────────────────────────────────────────────────────┐
│  NiFi (Ingestion + Enrichment) → Kafka → Flink (Real-time)  │
│       ↓                                    ↓                 │
│  100+ processors                     Stream processing      │
│       ↓                                    ↓                 │
│  10 topics (RF=3)                   enriched → 4+ consumers  │
│                                            ↓                 │
│                                   PG / ES / SIEM / Analytics │
└─────────────────────────────────────────────────────────────┘
```

**Characteristics:**
- ✅ HA (3 Kafka brokers, RF=3)
- ✅ Schema Registry for evolution
- ✅ Real-time processing (Flink)
- ✅ Comprehensive security (Vault, RBAC, TLS)
- ✅ Full observability (Prometheus + Grafana)
- ⚠️ More complex deployment
- ⚠️ Higher resource requirements

### 7.3 Pattern Comparison

| Aspect | v4.2 (Actual) | DCIM-Wiki (Reference) | Recommendation | Status |
|--------|---------------|----------------------|----------------|--------|
| **Ingestion** | Telegraf | NiFi | Keep Telegraf (simpler) | ✅ Keep |
| **Kafka** | 3 brokers, RF=3 | 3 brokers, RF=3 | ✅ **ALIGNED** | ✅ **DONE** |
| **Schema** | Schema Registry (Avro) | Schema Registry | ✅ **ALIGNED** | ✅ **DONE** |
| **Enrichment** | NiFi + Redis | NiFi + Redis | Keep (match) | ✅ Keep |
| **Processing** | Batch/near-RT | Real-time + batch | Add Flink for P1 alerts (P3) | ❌ TODO |
