## 2. Key Change: NiFi Cutover

### 2.1 What Changed

| Component | v4.3 (Before) | v4.4 (After) |
|-----------|---------------|---------------|
| **Server Redfish** | Telegraf `inputs.redfish` | **NiFi ExecuteProcess** (`redfish_poller.py`) |
| **Server Telemetry** | Telegraf `inputs.exec` | **NiFi ExecuteProcess** (`redfish_telemetry_poller.py`) |
| **UPS SNMP** | NiFi ExecuteProcess | NiFi ExecuteProcess (unchanged) |
| **NAS SNMP** | Telegraf `inputs.snmp` | **NiFi ExecuteProcess** (`nas_poller.py`) |
| **Network SNMP** | NiFi ExecuteProcess | NiFi ExecuteProcess (unchanged) |
| **CCTV ISAPI** | Standalone daemon (`dcim-cctv-poller.service`) | **NiFi ExecuteProcess** (`hikvision_poller.py`) |
| **Server Inventory** | Cron script → Kafka | **NiFi ExecuteProcess** (daily 01:00) |

### 2.2 Telegraf Status

| Telegraf Component | v4.3 Status | v4.4 Status |
|--------------------|-------------|-------------|
| `telegraf.service` | ✅ Active | ⚠️ **Active (system metrics only)** |
| `servers-redfish.conf` | ✅ Active | ❌ **Deactivated** |
| `nas-snmp.conf` | ✅ Active | ❌ **Deactivated** |
| `ups-apc.conf.disabled` | ❌ Disabled | ❌ Disabled |
| `mikrotik-snmp.conf` | ❌ Disabled | ❌ Disabled |
| `cctv-hikvision.conf` | ❌ Disabled | ❌ Disabled |
| `infra-monitoring.conf` | ✅ Active (ES/PG health) | ✅ Active (unchanged) |

**Telegraf sekarang hanya untuk:**
- System self-metrics (CPU, memory, disk) → `dcim-infra-metrics-*`
- Infrastructure monitoring (ES health, PG stats)

### 2.3 Why NiFi Cutover

| Reason | Explanation |
|--------|-------------|
| **Centralized management** | Semua collection di satu tempat (NiFi) |
| **Consistent scheduling** | NiFi processor scheduling lebih fleksibel |
| **Error handling terintegrasi** | NiFi built-in retry + DLQ routing |
| **Visual monitoring** | NiFi UI untuk memantau flow |
| **Alignment dengan reference** | DCIM-Wiki merekomendasikan NiFi sebagai ingestion tool |
